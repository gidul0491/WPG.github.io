컨테이너

종류 설명
.container 기본 container, 각 중단점 사이마다 일정한 너비를 가짐
.container-sm 화면 크기가 576px 이상이면 여백 존재
.container-md 화면 크기가 768px 이상이면 여백 존재
.container-lg 화면 크기가 992px 이상이면 여백 존재
.container-xl 화면 크기가 1200px 이상이면 여백 존재
.container-xl 화면 크기가 1400px 이상이면 여백 존재
.container-fluid 항상 여백이 없이 화면의 너비만큼의 크기를 가짐

버튼

종류 설명
.btn
.btn-link <a> 태그 링크에 사용
.btn-primary
.btn-outline-primary
.btn-sm || lg
.active
.btn-close X 모양의 닫기 버튼
.btn-group 버튼을 감싸는 div 태그에 적용시 버튼이 붙어있는 형태가 적용됨
.btn-group-vertical 버튼을 감싸는 div 태그에 적용시 버튼이 세로로 븥어있는 형태가 적용됨
.dropdown-toggle 드랍다운 버튼, 해당 클래스와 data-bs-toggle="dropdown" 속성을 버튼에 적용하고, 그 뒤에 <ul> 태그에 .dropdown-menu, <a>태그를 포함한 <li>태그에 .dropdown-item 클래스를 적용시켜 드랍다운 메뉴를 만들 수 있음
.dropdown-divider 드랍다운 메뉴 리스트의 li 태그에 hr태그를 구분선으로 만들어줌
.dropdown-header 드랍다운 메뉴 리스트의 li 태그 내의 h#태그 등을 회색 글씨의 선택 불가능한 헤더가 만들어짐
.dropdown-item-text 드랍다운 메뉴 리스트의 li 태그 내의 span태그 등을 선택 불가능한 일반 글자로 만들어줌
.active || disabled 드랍다운 메뉴 리스트의 li 태그 내의 a태그를 활성화/비활성화 상태로 만들어줌
.dropdown-menu-방향 방향에 start 또는 end를 넣고 드랍다운 버튼과 ul을 감싸는 div태그에 적용시키면 드랍다운 메뉴의 위치를 왼쪽/오른쪽으로 설정가능
.dropup 드랍다운 버튼과 ul을 감싸는 div태그에 적용시키면 드랍다운 메뉴가 나타날 때 위치를 아래가 아닌 위로 설정가능
.pagination a.page-link를 감싸는 li.page-item을 감싸는 ul태그에 .pagination을 적용시 페이지를 나타내주는 버튼 그룹이 됨, .disbled, .active 적용가능, .pagination-sm || lg 추가시 사이즈 설정 가능

디스플레이

종류 설명
.d-inline block 타입을 inline 타입으로 변경
.d-block inline 타입을 block 타입으로 변경
.m방향-크기 마진, 방향은 t(top), b(bottom), s(start, 왼쪽), e(end, 오른쪽), x(x축), y(y축)가 있으며 생략하면 모든 방향에 적용. 크기는 0, 1(0.25rem), 2(0.5rem), 3(1rem), 4(2rem), 5(3rem)
.p방향-크기 패딩, 방향과 크기는 .m- 과 동일한 방식으로 적용
.border 1px의 회색 테두리
.border-방향 방향은 top, bottom, start, end 가 있으며 생략시 전방향
.border-두께 두께는 0, 1, 2, 3, 4, 5 가 있으며 단위는 px, 0은 테두리 제거 효과가 있음, 생략시 1
.border-색상 테두리 색상 지정가능
.rounded-방향 지정한 방향의 테두리 모서리 2개를 둥글게 설정, 방향은 top, bottom, start, end가 있으며 생략시 전방향
.rounded-circle || pill circle은 테두리를 깎아 요소를 원모양으로 만들고 pill은 요소가 직사각형일시 알약 모양으로 만듦
.rounded-숫자 0, 1, 2, 3, 4, 5의 숫자를 사용할 수 있으며 숫자가 커질수록 테두리가 더욱 둥글어짐, 0은 테두리 제거
.clearfix 자식 요소에 float 속성을 줄 때 부모 요소에 .clearfix를 사용하면 자식 요소들끼리 겹치거나 하는 문제를 없애줌
.float-start 요소에 float-left 속성을 부여
.float-end 요소에 float-right 속성을 부여
.invisible 요소가 실제로는 존재하지만 시각적으로 보이지 않게 해줌
.visible 요소가 보이게 해줌
.w-숫자 부모요소 대비 상대적 너비(width), 25, 50, 75, 100으로 설정 가능
.w-auto width: auto와 같은 속성, block level 요소의 경우 부모요소의 너비만큼, inline-block 요소의 경우 자식요소의 너비만큼으로 조절됨
.mw-100 max-width:100%와 같은 속성, 자식 요소의 크기가 커서 부모 요소의 너비보다 더 커지는 경우를 방지할 수 있음
.h-숫자 부모요소 대비 상대적 높이(height), 25, 50, 75, 100으로 설정 가능
.h-auto height: auto와 같은 속성, block 요소의 경우 부모요소의 높이만큼, inline-block 요소의 경우 자식요소의 높이만큼으로 조절됨
.mg-100 max-height:100%와 같은 속성, 자식 요소의 크기가 커서 부모 요소의 높이보다 더 커지는 경우를 방지할 수 있음
.shadow-크기 그림자 효과 부여, 크기는 none, sm, 생략, lg 가 있으며 none은 그림자 효과 제거, 생략은 중간 크기의 그림자 효과
.ratio ratio-가로비율x세로비율 <iframe>, <embed>, <video>, 그리고 <object>와 같은 외부 콘텐츠의 부모 요소에 클래스를 지정하여 컨텐츠의 가로세로 비율을 설정 가능, 1x1, 4x3, 16x9, 21x9 중에서 선택 가능

.nav a.nav-link를 감싸는 li.nav-item을 감싸는 ul 태그에 .nav 적용시 내비게이션 메뉴 형성, 플렉스 컨테이너이므로 li 태그들을 정렬할 수 있음

.navbar 내비게이션 바, nav or div 태그에 .navbar, navbar-expand-크기를 추가하고 자식요소로 ul.navbar-nav, 그 안에 li.nav-item, 그 안에 a.nav-link를 추가. .navbar-expand-크기 클래스 제거시 li가 세로로 정렬됨
.navbar-dark 내비게이션 바의 글씨가 밝은색으로 바뀜, 최상위 nav 태그에 사용
.navbar-brand 웹 페이지의 메인 아인콘, nav.navbar 내의 ul.navbar-nav 앞에 a.navbar-brand 추가시 원하는 링크로 이동하는 브랜드가 생성됨, img 태그를 자식 태그로 추가하여 이미지도 사용 가능
.navbar-text 아무 기능이 없는 텍스트, nav.navbar 내의 ul.navbar-nav의 형제 요소로 span.navbar-text를 추가
.navbar-toggler
화면이 작으면 내비게이션 메뉴들이 사라지고 하나의 버튼을 눌러야 메뉴들이 보이게되는 내비게이션 바. .nav-navbar 내에 a.navbar-brand 등과 형제요소로서 button.navbar-toggler에 data-bs-toggle="collapse", data-bs-target="#ul.navbar-nav를 감싸는 div의 id" 를 추가하고 button태그 안에 span.navbar-toggler-icon을 넣고, button.navbar-toggler와 형제요소로서 div.coppapse.navbar-collapse#ul.navbar-nav를 감싸는 div의 id" 를 추가하고 그 안에 ul.nav-item을 넣어 내비게이션 메뉴 삽입. nav.navbar.navbar-expand-크기 태그에서 크기에 따라 어느 정도의 크기에서 메뉴를 감출 것인지 설정 가능
.fixed-위치 내비게이션 바를 화면상에 고정시켜줌. 위치는 top 또는 bottom 사용 가능, 내비게이션 바의 최상위 부모의 클래스로 사용
.sticky-bar 내비게이션 바가 처음엔 웹페이지의 중간에 있더라도 스크롤을 내리면 화면 위쪽 밖으로 넘어가 안보이는 것이 아닌 회면 최상단에 붙어서 화면과 같이 움직이게 됨, 내비게이션 바의 최상위 부모의 클래스로 사용
data-bs-spy="scroll"
페이지 내의 현재 스크롤 위치에 따라 내비게이션 바의 특정 버튼이 활성화되는 기능 html문서의 body 태그의 스타일로 position:relative 속성을, 사용자 속성으로 data-bs-spy="scroll", data-bs-target="네비게이션 바의 클래스", data-bs-offset="50(수정가능)" 속성을 추가, navbar를 화면 상단 고정. li 태그 내의 링크가 페이지의 특정 영역과 id로 연결되어 있다면, 페이지의 특정 영역이 화면에 들어올 때 해당 영역과 연결된 내비게이션 바의 링크가 활성화 된 것으로 표시됨

캐러셀

종류 설명
.carousel 여러 장의 이미지가 자동으로 넘어가는 캐러셀 슬라이드, 최상위 부모로 div#캐러셀id.carousel.slide 를 설정하고 그 안에 div.carousel-indicators, div.carousel-inner, button.carousel-control-prev, button.carousel-control-next가 포함됨
.carousel-indicators indicator, 캐러셀 내의 이미지가 몇 번째 이미지인지를 나타내줌. div.carousel-indicators 안에 button type="button" 태그를 추가하고 사용자 속성으로 data-bs-target="#캐러셀id" data-bs-slide-to="n(0부터 시작하는 정수)"를 추가. 첫 번째 버튼은 .active를 추가
.carousel-inner slideshow, 캐러셀 내에 이미지 등을 포함하는 div.darousel-item을 슬라이드쇼로 만들어줌
.darousel-item 캐러셀의 내용물. img와 같은 캐러셀 아이템을 블럭 타입으로 넣어주고 첫 번째 슬라이드는 .active를 추가해야함. data-bs-interval="시간" 속성을 추가하면 캐러셀 아이템이 전환되기까지의 시간을 설정가능하고 시간 단위는 ms
.carousel-control-방향 캐러셀 아이템 넘길 수 있는 버튼, 방향은 prev 와 next가 있음. div.carousel의 자식요소로 button type="button"을 추가하고 .carousel-control-방향 클래스, data-bs-target="#캐러셀id" data-bs-slide="방향" 속성을 추가. 그 안에 span.carousel-control-prev/next/공백-icon을 추가
.carousel-caption 캐러셀 아이템 위에 캡션을 추가. div.carousel-item의 자식요소로 div.carousel-caption을 추가하고 내용을 쓰면 됨
.carousel-fade fade in/out 방식으로 이미지 전환, 캐러셀 최상위 부모 요소에 추가.

모달

종류 설명
data-bs-toggle="modal" 모달창을 열기위한 버튼, button 태그에 data-bs-toggle="modal", data-bs-target="#모달id" 속성을 추가
.modal 모달 요소들의 최상위 부모 div에 적용, #모달id도 추가되어야 함. 모달창이 열릴 때 배경을 어둡게 만들어주는 요소
.modal-dialog 모달창으로 표시되는 영역, div.modal의 자식요소로 div.modal-dialog를 넣어주어야 함
.modal-content 모달창, div.modal-dialog의 자식요소로 넣어주어야 함
.modal-header 모달창 헤더 영역, div.modal-content의 자식요소로 넣어주어야 함
.modal-body 모달창의 바디 영역, div.modal-content의 자식요소로 넣어주어야 함
.modal-footer 모달창 푸터 영역, div.modal-content의 자식요소로 넣어주어야 함
.modal-title 모달창의 타이틀, h# 태그 등에 사용됨
data-bs-dismiss="modal" 모달창을 끄는 버튼, button 태그에 사용됨
.modal-크기 모달창 크기 설정, div.modal-dialog 요소에 .modal-크기 를 추가하여 모달창 사이즈 설정 가능. 크기는 sm, lg, xl, fullscreen이 있음
.modal-dialog-centered 모달창을 가운데로 띄우게 함, div.modal-dialog 요소에 추가
.modal-dialog-scrollable 모달창 내에서 스크롤이 가능하게 함, div.modal-dialog 요소에 추가

오프캔버스

종류 설명
data-bs-toggle="offcanvas" 오프캔버스를 열기위한 버튼, button 태그에 data-bs-toggle="offcanvas", data-bs-target="#오프캔버스id" 추가
.offcanvas 오프캔버스 div 설정, id를 반드시 설정해주어야 함
.offcanvas-위치 오프캔버스가 열리는 위치 설정, 위치는 start, end, top, bottom이 있음
.offcanvas-header 오프캔버스의 헤더영역 지정, div태그에 적용
.offcanvas-title 텍스트를 오프캔버스의 제목으로 지정
data-bs-dismiss="offcanvas" 오프캔버스를 닫는 버튼, button 태그에 속성으로 추가
.offcanvas-body 오프캔버스의 바디 영역을 지정
.offcanvas-body 오프캔버스의 바디 영역을 지정
data-bs-scroll="true" 오프캔버스를 스크롤 가능하게 지정

플렉스

종류 설명
.d-flex 부트스트랩의 클래스를 이용한 플렉스, div에 적용하여 플렉스 컨테이너로 지정
.d-inline-container 플렉스 컨테이너를 인라인 방식으로 만들어줌, 블럭 방식과 달리 한줄 전체를 사용하지 않음
.flex-row 플렉스 컨테이너 내의 플렉스 항목들을 가로로 배치
.flex-row-reverse 플렉스 컨테이너 내의 플렉스 항목들을 가로로 배치하고 역순으로 배열
.flex-column 플렉스 항목을 세로로 배치
.flex-column-reverse 플렉스 항목을 세로로 배치하고 역순으로 배열
.justify-content-정렬방식 플렉스 컨테이너에 적용하는 플렉스 항목들의 가로방향 정렬 방식, start, center, end, between, around가 있음
.align-content-정렬방식 플렉스 컨테이너에 적용하는 플렉스 항목들의 세로방향 정렬 방식, start, center, end, around, stretch가 있음
.align-self-정렬방식 플렉스 항목에 적용하는 세로방향 정렬 방식, start, center, end, baseline, stretch가 있음
.flex-fill 플렉스 항목을 빈공간을 채우도록 늘림, 플렉스 항목에 적용
.flex-grow-1 플렉스 항목을 빈공간을 채우도록 늘림, 플렉스 항목에 적용
.order-# 플렉스 항목의 배치 순서를 #으로 지정
.flex-nowrap 플렉스 항목이 플렉스 컨테이너를 벗어날 수 있게 설정
.flex-wrap 플렉스 항목이 플렉스 컨테이너를 벗어날 수 없으며 항목이 많을 시 줄이 바뀜
.flex-wrap-reverse 플렉스 항목이 플렉스 컨테이너를 벗어날 수 없으며 항목이 많을 시 줄이 위로 바뀜
