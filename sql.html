셀렉트 문

종류 설명
SELECT 조회할 열 이름/데이터를 하나 이상 지정, *은 모든 열을 의미
FROM 조회할 테이블 이름, SELECT * FROM emp
DISTINCT SELECT절에 명시한 열에서 중복 값을 제외해줌, SELECT DISTINCT * FROM emp
ALL SELECT절에 명시한 열에서 중복 값을 제외 없이 모두 표시해줌, SELECT ALL * FROM emp
AS
별칭 지정, 별칭의 지정 방법에는 4가지가 있음 1. -data- 별칭
2. -data- "별칭"
3. -data- AS 별칭
4. -data- AS "별칭"
이 중 3번 방법이 가장 좋음
ORDER BY 정렬 기준이 되는 열을 선택할 수 있음, SELECT * FROM emp ORDER BY sal (DESC/ASC)

WHERE, 연산자

종류 설명
WHERE 특정 조건을 기준으로 원하는 행을 출력, SELECT * FROM emp WHERE deptno = 30;
AND AND 논리 연산자 SELECT * FROM emp WHERE deptno = 30 AND job = 'SALESMAN';
OR OR 논리 연산자 SELECT * FROM emp WHERE deptno = 30 OR job = 'SALESMAN';
+, -, *, / 산술 연산자, 나눗셈 결과는 실수로 출력됨
>, >=, <, <= 비교 연산자
=, !=, <>, ^= 등가 비교 연산자, A = B 는 A와 B가 같을 경우 TRUE, A (!=, <>, ^=) B는 A와 B가 다를 경우 TRUE
NOT 값 논리 부정 연산자, 값이 아닌 경우에만 TRUE, SELECT * FROM emp WHERE NOT sal = 3000;
IN 동일한 열의 조건이 여러 가지일 경우 OR 연산자로 묶는 것보다 편리한 방법, SELECT * FROM emp WHERE job IN ['manager', 'salesman', 'clerk'];
BETWEEN A AND B A 이상 B 이하인 경우에 TRUE, SELECT * FROM emp WHERE sal BETWEEN 2000 AND 3000;
LIKE 일부 문자열이 포함된 데이터를 조회 WHERE ename LIKE '_L%'
_ , % 와일드 카드, _는 하나의 문자열, %는 길이와 상관없는 문자열을 의미, 'S%', '_L%' 등
\ 이스케이프 문자, 와일드카드와 같은 특수문자 앞에 붙여주면 와일드카드를 단순한 문자로 인식함
IS NULL 데이터 값이 NULL인 경우 비교 연산이 불가능하기 때문에 값이 NULL인지 구분하기 위해서는 A = NULL 이 아니라 A IS NULL 이라고 써야함
UNION (ALL) 집합 연산자, 두 SELECT문을 합쳐서 하나의 결과로 보여줌, 두 SELECT문의 열 개수와 열의 자료형이 같아야함, UNION ALL은 중복값도 모두 포함해서 출력
MINUS 먼저 작성한 SELECT문의 결과에서 뒤에 작성한 SELECT 문의 결과들을 뺀 결과만 보여줌
INTERSECT 먼저 작성한 SELECT 문과 뒤에 작성한 SELECT 문의 결과의 교집합만 보여줌
연산자 우선순위 *, / > +, - > =, !=, <>, ^=, >, >=, <, <= > IS (NOT) NULL, (NOT) LIKE, (NOT) IN > BETWEEN A AND B > NOT > AND > OR

문자 함수

종류 설명
UPPER(문자열) 문자 데이터를 모두 대문자로 변환, SELECT UPPER(ename) FROM emp WHERE deptno = 30;
LOWER(문자열) 문자 데이터를 모두 소문자로 변환, SELECT * FROM emp WHERE LOWER(ename) LIKE LOWER();
INITCAP(문자열) 문자 데이터를 첫 글자는 대문자로, 나머지는 소문자로 변환, SELECT INITCAP(ename) FROM emp;
LENGTH(문자열) 문자열의 길이를 구하는 함수, SELECT LENGTH(ename) FROM emp;
LENGTHB(문자열) 문자열의 바이트 크기를 구하는 함수, LENGTH() 함수는 한글의 길이를 1로 계산하지만 LENGTHB()는 2로 계산함, SELECT ename FROM emp WHERE LENGTHB(ename) >= 2;
SUBSTR(문자열, 시작위치 , 추출 길이) 문자열 일부를 추출, 시작위치가 음수일 경우 마지막 문자에서부터 왼쪽으로의 위치, 추출 길이를 생략시 시작위치부터 오른쪽 끝까지 출력 SELECT SUBSTR(job,1,2) SUBSTR(job,5) FROM emp;
INSTR(문자열, 찾을 문자열 , 탐색을 시작할 문자열 위치, 몇 번째 문자열을 찾을지 ) 문자열에서 찾고싶은 문자 또는 문자열이 몇 번째에 있는지를 알려주는 함수, SELECT INSTR('HELLO, ORACLE!','L',2,2) AS INSTR FROM DUAL;
REPLACE(문자열, 찾을 문자열 , 대체할 문자 ) 문자열에서 찾고싶은 문자 또는 문자열을 대체하거나 제거하는 함수, SELECT REPLACE('010-1234-5678','-',' ') AS REPLACE FROM DUAL;
LPAD, RPAD(문자열, 자릿수 , 빈 공간에 채울 문자 ) 문자열을 입력한 자릿수만큼의 길이로 만들고 빈 공간은 공백 또는 원하는 문자로 LPAD는 왼쪽, RPAD는 오른쪽에 채워넣음, SELECT RPAD('971225-', 14, '*') FROM DUAL;
CONCAT(문자열1, 문자열2) 문자열1 뒤에 문자열2를 붙여줌 SELECT CONCAT(empno, CONCAT(' : ', ename)) FROM emp;
TRIM(삭제옵션 삭제할문자 FROM 문자열) 문자열의 시작/끝부분을 잘라주는 함수, 삭제옵션은 LEADING, TRAILING, BOTH가 있음, 삭제할 문자는 적지 않으면 공백이 제거되고 적고싶으면 ''안에 입력, SELECT TRIM(LEADING '_' FROM '__ORACLE__') FROM DUAL;
LTRIM, RTRIM(문자열 , 삭제할 문자) 문자열의 시작/끝부분을 잘라주는 함수, 삭제할 문자는 적지 않으면 공백이 제거되고 적고싶으면 ''안에 입력, ''안의 모든 문자를 대상 문자열의 시작/끝부분에서 지워준다고 보면 됨 SELECT LTRIM('_/_/ORACLE/_/_', '/_/) FROM DUAL; 결과 : ORACLE/_/_

숫자 함수

종류 설명
ROUND(숫자, 반올림하여 나타낼 소수점 위치) 숫자를 반올림, 소수점 위치를 지정하지 않으면 정수로 표현됨, 위치는 실수의 첫째자리수까지 나타내는 것을 0으로 생각하고 소수부에서 반올림하고 싶으면 양수, 정수부에서 반올림하고 싶으면 음수를 사용
TRUNC(숫자, 버림하여 나타낼 소수점 위치) 숫자를 버 림, 소수점 위치를 지정하지 않으면 정수로 표현됨, 위치는 실수의 첫째자리수까지 나타내는 것을 0으로 생각하고 소수부에서 버림하고 싶으면 양수, 정수부에서 버림하고 싶으면 음수를 사용
CEIL(숫자) 입력한 숫자보다 크면서 가장 작은 정수를 출력
FLOOR(숫자) 입력한 숫자보다 작으면서 가장 큰 정수를 출력
MOD(나눠질숫자, 나눌숫자) 나머지를 구하는 함수

날짜 함수

종류 설명
SYSDATE 오라클 데이터베이스 서버가 놓인 OS의 날짜와 시간 표시, SYDATE+1이면 하루 뒤, -1이면 하루 전 날짜
ADD_MONTHS(날짜, 정수) 입력한 날짜에서 정수만큼의 개월 수 뒤의 날짜를 출력
MONTHS_BETWEEN(날짜1, 날짜2) 날짜2로부터 날짜1까지 흐른 개월 수를 계산후 출력, TRUNC를 이용하여 정수부만 출력하면 좋음
NEXT_DAY(날짜, 요일) 날짜를 기준으로 바로 다음으로 오게되는 해당 요일의 날짜 SELECT TO_CHAR(NEXT_DAY(ADD_MONTHS(hiredate,3),'월'),'YYYY-MM-DD') AS R_JOB
ROUND, TRUNC(날짜, 날짜 포맷)
날짜를 날짜 포맷에 맞추어 출력
날짜 포맷 종류 설명
CC, SCC 세기
SYYYY, YYYY, YEAR, SYEAR, YYY, YY, Y 해당 데이터의 7월 1일부터는 다음 연도인 것으로 반올림
RRRR,RR 연을 4자리, 2자리로 표시
IYYY, IYY, IY, I 2009, 009, 09, 9로 표시, ISO 8601 기준
Q 분기, 3개월이 1분기이므로 2번째 달의 16일부터는 다음 분기인 것으로 반올림
MONTH, MON, MM, RM NOVEMBER, NOV, 11, IX로 달을 표시, 각 달의 16일부터는 다음 달인 것으로 반올림
WW 해당 연도의 몇 번째 주인지 표시
IW 해당 연도의 몇 번째 주인지 표시, ISO 8601 기준
W 해당 월의 몇 번째 주인지 표시
DDD, DD, J 해당 연도에서 몇 번째 일인지, 해당 달에서 몇 번째 일인지, Julian 년호 숫자, 정오를 기준으로 반올림
DAY, DY, D FRIDAY, FRI, 주중 일로 표시
HH, HH12, HH24 06, 06, 18로 시간을 표시
MI 분을 표시

다중행 함수

종류 설명
SUM(DISTINCT||ALL||공백 열) OVER(분석을 위한 문법)(생략가능) 지정한 열의 총합 반환
COUNT(DISTINCT||ALL||공백 열) OVER(분석을 위한 문법)(생략가능) 지정한 열의 데이터 개수 반환
MAX(DISTINCT||ALL||공백 열) OVER(분석을 위한 문법)(생략가능) 지정한 열의 최댓값 반환, 날짜데이터도 사용가능
MIN(DISTINCT||ALL||공백 열) OVER(분석을 위한 문법)(생략가능) 지정한 열의 최솟값 반환, 날짜데이터도 사용가능
AVG(DISTINCT||ALL||공백 열) OVER(분석을 위한 문법)(생략가능) 지정한 열의 평균값 반환
GROUP BY 다중행 함수 사용시 그룹이 없으면 모든 행에 대하여 합, 평균 등을 구하게 됨. 하지만 GROUP BY로 다중행함수로 계산되지 않는 열을 묶어주면 각 행마다 계산결과가 표시됨. SELECT AVG(sal), deptno FROM emp GROUP BY deptno; 결과: 각 deptno마다 sal의 평균값이 구해짐
HAVING
GROUP BY 절이 있을 때 사용가능, 다중행 함수의 결과값에 조건을 걸어 결과물을 출력, WHERE이 각 행의 데이터에 조건을 거는 것이라면 HAVING은 각 행의 데이터를 이용하여 다중행 함수로 그룹별 결과들을 얻었을 때 그 결과들에 대해 조건을 거는것
  • SELECT deptno, job, AVG(sal)
  • FROM emp
  • GROUP BY deptno, job
  • HAVING AVG(sal) >= 2000
  • ORDER BY deptno, job
ROLLUP(열1, 열2, ...) GROUP BY 바로 다음에 사용가능, GROUP BY만 사용하면 열1과 열2의 조합별로 다중행함수가 실행되지만 ROLLUP 사용시 열1과 열2의 조합별 뿐만아니라 열2는 구분하지않고 열1만으로 구분된 다중행함수 실행결과, 그리고 열1로도 구분되지않은 모든 데이터에 대한 다중행함수 실행결과가 출력됨
예시) ROLLUP(A, B, C)
1. A그룹별 B그룹별 C그룹에 해당하는 결과 출력
2. A그룹별 B그룹에 해당하는 결과 출력
3. A그룹에 해당하는 결과 출력
4. 전체 데이터 결과 출력
CUBE(열1, 열2, ...) GROUP BY 바로 다음에 사용가능, GROUP BY만 사용하면 열1과 열2의 조합별로 다중행함수가 실행되지만 CUBE 사용시 열1과 열2의 조합별 뿐만아니라 열2는 구분하지않고 열1만으로 구분된 다중행함수 실행결과, 열1는 구분하지않고 열2만으로 구분된 다중행함수 실행결과, 그리고 구분되지않은 모든 데이터에 대한 다중행함수 실행결과가 출력됨
예시) ROLLUP(A, B, C)
1. A그룹별 B그룹별 C그룹에 해당하는 결과 출력
2. A그룹별 B그룹에 해당하는 결과 출력
3. B그룹별 c그룹에 해당하는 결과 출력
4. A그룹별 C그룹에 해당하는 결과 출력
5. A그룹에 해당하는 결과 출력
6. A그룹에 해당하는 결과 출력
7. A그룹에 해당하는 결과 출력
8. A그룹에 해당하는 결과 출력
9. 전체 데이터 결과 출력
GROUPING SETS(열1, 열2, ...) GROUP BY 바로 다음에 사용 가능, 열1로 그룹화하여 다중항함수를 실행하고, 그다음으로 열2로 그룹화하여 다중행함수를 실행. 열1을 그룹화했을때의 결과와 열2를 그룹화했을때의 결과를 그냥 붙여서 출력하는 것으로 볼 수 있음
GROUPING(열1, 열2, ...) SELECT 절에 사용가능, 그룹화한 열1, 열2, ...를 선택하면 해당 열이 그룹화가 된 것인지 아닌지를 표시해줌, 그룹화된 열은 0, 안된 열은 1로 표시
GROUPING_ID(열1, 열2, ...) SELECT 절에 사용가능, 각 열이 그룹화되었는지 아닌지에 따라 GROUPING 함수는 열별로 1또는 0으로 표시해주는데 이를 2진수 취급하여 10진수로 나타내 주는 것이 GROUPING_ID 함수
LISTAGG(열, '구분자') WITHIN GROUP(ORDER BY 정렬기준이 되는 열) SELECT 절에 사용가능, 열을 세로로 출력하는 것이 아닌 GROUP BY로 묶여진 열로 구분되어 가로로 구분됨
예시
  • SELECT deptno,
  • LISTAGG(enames, ',')
  • WITHIN GROUP(ORDER BY SAL DESC) AS enames
  • FROM emp
  • GROUP BY deptno;
deptno enames
10 king, clark, miller
20 ford, scott, jones, adams, smith
30 blake, allen, turner, martin, ward, james

  • ORDER BY y축 구분 열;

  • ORDER BY ...

조인

종류 설명
WHERE 테이블명칭1.외래키1 = 테이블명칭2.주키2
등가조인, 다른 테이블을 조인하는 방법 중 한 가지
  • SELECT E.empno, E.ename, D.deptno, ...
  • FROM emp E, dept D
  • WHERE E.deptno = D.deptno
  • ORDER BY ...;
WHERE 테이블명칭1.외래키1 BETWEEN 테이블명칭2.기준열1 AND 테이블명칭2.기준열2
비등가조인, 다른 테이블을 조인하는 방법 중 한 가지
  • SELECT *
  • FROM emp E, salgrade S
  • WHERE E.sal BETWEEN S.losal AND S.hisal;
salgrade 테이블이 grade, losal, hisal 열로 이루어져 있을 때 emp의 sal의 한 데이터가 losal과 hisal의 범위 내에 있을 때 그 행은 해당 grade 열의 데이터를 가져가게 됨
WHERE 같은테이블의명칭1.외래키1 = 같은테이블의명칭2.주키2
자체조인, 같은 테이블에서 서로 다른 두 행을 붙여야 할 때 사용
  • SELECT *
  • FROM emp E1, emp E2
  • WHERE E1.mgr = E2.empno;
WHERE 절에서 (해당 테이블에서 외래키로 사용될 열) = (해당 테이블의 주키) 형태를 취함
WHERE 테이블명칭1.외래키1(+)= 테이블명칭2.주키2(+) 왼쪽 또는 오른쪽(다른 테이블을 null값으로 불러오는 기준이 되는 쪽) 외부조인, (+)가 붙어있는 쪽의 테이블은 다른 테이블의 외래키에 맞는 데이터가 없더라도 NULL값으로서 모조리 가져옴
NATURAL JOIN
등가조인, 기준열이 되는 열 앞에 테이블명이 붙으면 안됨
  • SELECT E.empno, deptno, D.dname
  • FROM emp E NATURAL JOIN dept D
기존의 WHERE 절이 필요없어짐
JOIN USING
등가조인, 기준열이 되는 열 앞에 테이블명이 붙으면 안됨
  • SELECT E.empno, deptno, D.dname
  • FROM emp E JOIN dept D USING (deptno)
기존의 WHERE 절이 필요없어짐
JOIN ON
가장 범용성 있는 조인, 기존의 WHERE 절에 있는 조인 조건식을 ON 옆에 작성
  • SELECT E.empno, E.deptno, D.dname
  • FROM emp E JOIN dept D ON (E.deptno = D.deptno)
OUTER JOIN
외부조인, 기존의 WHERE 절에 있는 조인 조건식을 ON 옆에 작성
  • SELECT E1.ename, E1.mgr, E2.ename
  • FROM emp E1 LEFT||RIGHT||FULL OUTER JOIN emp E2 ON (E1.mgr = E2.empno)

서브쿼리

종류 설명
ANY, SOME
서브쿼리가 반환한 값 중 하나라도 메인쿼리와의 조건식 결과가 TRUE라면 메인쿼리 조선식을 TRUE로 반환
  • SELECT *
  • FROM emp
  • WHERE sal = SOME||ANY (SELECT MAX(sal) FROM emp GROUP BY deptno);
deptno 그룹별로 sal의 최댓값을 가지는 행들의 모든 데이터가 출력됨
ALL
서브쿼리가 반환한 모든 값들과 메인쿼리와의 조건식 결과가 TRUE라면 메인쿼리 조선식을 TRUE로 반환
  • SELECT *
  • FROM emp
  • WHERE sal < ALL (SELECT sal FROM emp WHERE deptno = 30);
deptno이 30인 모든 행의 sal값보다 작은 sal 값을 가지는 행의 모든 데이터가 출력됨
EXIST
서브쿼리가 반환한 하나라도 TRUE라면 메인쿼리 조선식을 TRUE로 반환
  • SELECT *
  • FROM emp
  • WHERE EXIST (SELECT dname FROM dept WHERE deptno = 10);
deptno이 10인 dname (FROM dept) 값이 하나라도 존재하면 모든 데이터를 출력
WITH
큰 테이블을 나눠서 작은 테이블로 만들어 데이터를 추출하기 위한 방법, FROM 절에 서브쿼리를 작성하여 불러오는 것보다 속도가 빠름
  • WITH
  • E10 AS (SELECT * FROM emp WHERE deptno = 10)
  • D AS (SELECT * FROM dept)
  • SELECT E10.empno, D.dname, ...
  • FROM E10, D
  • WHERE E10.DEPTNO = D.DEPTNO;

특정 테이블의 열을 참조하는 자료형, 지정한 테이블의 열과 같은 자료형이 됨 변수이름 테이블이름.열이름%TYPE


DELCARE
V_DEPTNO DEPT.DEPTNO%TYPE := 50;
BEGIN
DBMS_OUTPUT.PUT_LINE('V_DEPTNO : ' || V_DEPTNO);
END;
/
%ROWTYPE
특정 테이블의 행을 참조하는 자료형, 변수이름.열이름1, 변수이름.열이름2 와 같이 테이블의 열에 해당하는 값을 가짐 변수이름 테이블이름%ROWTYPE


DELCARE
V_DEPT_ROW DEPT%ROWTYPE;
BEGIN
SELECT DEPTNO, DNAME, LOC INTO V_DEPT_ROW;
FROM DEPT
WHERE DEPTNO = 40;
DBMS_OUTPUT.PUT_LINE('DEPTNO : ' || V_DEPT_ROW.DEPTNO);
DBMS_OUTPUT.PUT_LINE('DNAME : ' || V_DEPT_ROW.DNAME);
DBMS_OUTPUT.PUT_LINE('LOC : ' || V_DEPT_ROW.LOC);
END;
/

IF-THEN [-ELSIF-THEN] [-ELSE]
특정 상황일 때 명령어 수행 IF 조건식 THEN 수행할명령어 END IF;


DECLARE
V_NUMBER NUMBER := 13;
BEGIN
IF MOD(V_NUMBER, 2) = 1 THEN
DBMS_OUTPUT.PUT_LINE('V_NUMBER는 홀수');

ELSIF MOD(V_NUMBER, 2) = 0 THEN DBMS_OUTPUT.PUT_LINE('V_NUMBER는 짝수');

ELSE
DBMS_OUTPUT.PUT_LINE('V_NUMBER는 유리수');
END IF;
END;

WHEN 값1 THEN 명령어;
WHEN 값2 THEN 명령어;
...
ELSE
명령어;
END CASE;


DECLARE
V_SCORE NUMBER := 89;
BEGIN

CASE TRUNC(V_SCORE/10)
WHEN 10 THEN DBMS_OUTPUT.PUT_LINE('A학점');
WHEN 9 THEN DBMS_OUTPUT.PUT_LINE('A학점');
WHEN 8 THEN DBMS_OUTPUT.PUT_LINE('B학점');
WHEN 7 THEN DBMS_OUTPUT.PUT_LINE('C학점');
WHEN 6 THEN DBMS_OUTPUT.PUT_LINE('D학점');
ELSE DBMS_OUTPUT.PUT_LINE('F학점');
END CASE;

END;
